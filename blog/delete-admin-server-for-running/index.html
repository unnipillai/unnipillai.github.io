Delete Admin Server for a running Weblogic domain

Bootstrap a standby server first — then delete the original

UP
Unni Pillai March 24, 2010

There would be scenarios in prod/dev environments which warrants us to create a new / stand by AdminServer, you may follow the below gives steps to achive the same.

You cannot use the Administration Console to delete the Administration Server as the admin console runs on the Administration Server

Try these steps:

  1. Create a new server name: newAdminServer (say port number: 8001)

  2. Stop all running servers in the domain on which you are working on

  3. Start the new server as admin server: java -Dweblogic.Name= newAdminServer weblogic.Server

  4. Start the Administration Console for the newly-started server http://ip:8001/console

  5. Delete the default admin server form the console

Cheers :)
